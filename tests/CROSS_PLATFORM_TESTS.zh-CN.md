# 跨平台测试矩阵

本文件记录必须在 macOS、Windows、Linux 的 Mihomo TUN 环境中完成的实测。自动测试不能代替真实系统的路由、DNS 和防火墙行为。

## 执行顺序

1. 保存修改前的项目覆写作为 baseline；不要改动订阅文件。
2. 在同一设备、同一网络、相邻时间段交错运行 baseline 与 candidate，每组至少 10 次。
3. 分别测试 Light/Full、家庭网络/手机热点、IPv4-only/IPv6-only/正常双栈/IPv6 黑洞。
4. 用 `npm run benchmark` 采样，用 `npm run benchmark:compare` 执行 10% 中位数回归门槛。
5. 单独切换 `tcp-concurrent` 或 `unified-delay`；不得同时改变两个变量。只有三个平台均通过才可加入默认覆写。

## 必须通过的场景

| 类别 | 操作 | 通过条件 |
| --- | --- | --- |
| DNS | 国内、境外域名冷/热查询 | 境外查询不出现 ISP DNS；国内查询只使用指定国内加密 DNS |
| IPv6 | 四种栈状态分别访问国内外站点 | 无旁路；IPv6 黑洞能及时回退，敏感流量不直连 |
| 故障 | 阻断主 DNS、境外 DoH、当前节点或 UDP | 不死锁、不错误直连，恢复后无需重启配置 |
| 切网 | Wi-Fi 与热点切换、休眠唤醒、TUN 重启 | DNS 与默认路由恢复，切换窗口无泄漏 |
| 缓存 | 规则提供者离线、缓存过期、离线启动 | 有缓存时正常启动；无缓存时明确失败而非放行敏感流量 |
| 兼容 | STUN、局域网域名、系统联网检测、AI、GitHub、流媒体、游戏 | 规则命中预期组，应用功能正常 |
| 暴露面 | 从同一局域网另一设备探测代理端口 | 无法连接 |

## 性能记录

记录 DNS、请求首连的中位数/P95，并另外记录吞吐、CPU、内存、节点切换次数和故障恢复秒数。DNS、首连或吞吐中位数相对项目旧版下降超过 10% 即失败。任何泄漏都直接失败，不以性能收益抵消。

## 当前自动验证与实测边界

- `npm test`：构建一致性、规则/组引用、循环、重复规则、MATCH 顺序、IP `no-resolve`、DNS 出口和脱敏订阅兼容性。
- `npm run audit:subscription -- /path/to/file.yaml`：只读检查真实订阅与项目覆写的结构兼容性，不显示节点值。
- DNS 泄漏、WebRTC、系统防火墙、TUN 切网、吞吐、CPU 与内存必须在三平台人工或 CI 主机上实测。
